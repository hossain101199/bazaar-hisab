# Security Policy

## Reporting a Vulnerability

If a security vulnerability is found in this project, please **do not** open a public GitHub issue.

Instead, report it privately by emailing:

**fshossain10@gmail.com**

Include the following in the report:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

A response will be sent within **72 hours**. Once the issue is confirmed, a fix will be released as soon as possible.

## Scope

The following are considered in scope:

- Authentication bypass
- Authorization issues (accessing another user's data)
- SQL injection or database exposure
- Sensitive data exposure (tokens, passwords)
- XSS or CSRF vulnerabilities

## Out of Scope

- Issues in third-party dependencies (report directly to the dependency maintainer)
- Bugs that do not have a security impact
